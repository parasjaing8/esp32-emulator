  CONCEPT                                                                                                               
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  
  This app connects to ESP32 microcontroller boards over Bluetooth (BLE) and acts as a universal control panel. Think: Arduino IDE + Serial
  Monitor + Device Manager, but mobile-first and beautiful.

  The ESP32 runs two firmwares in separate OTA partitions:
  - Partition A = "OS firmware" — always present, exposes BLE control interface (GPIO, serial, OTA)
  - Partition B = "App firmware" — user's project code (WaterTank controller, LED driver, etc.)

  The analogy: ESP32 OS firmware = Windows. Project firmware = installed application.
  The user can flash any .bin to partition B via BLE OTA, then switch boot target — just like installing software on a PC.

  1. SIMULATION / DEMO MODE                                                                                             
  Since no real firmware exists yet, build a full simulation mode that auto-activates when no board is connected. Mock a connected ESP32-C3
  board with:
  - Random GPIO pin state changes every 2s (pins 0-10 fluctuate HIGH/LOW)
  - Simulated ADC readings (0-4095 on pins 1-4, slowly changing)
  - Serial output stream with fake log lines ("Boot complete", "Sensor read: 42", etc.)
  - Fake OTA progress bar when a file is "flashed"
  This lets the entire app be fully usable and testable without hardware.

  2. BOARD SCREEN (Tab 1 — "Board")
  - Clean card showing: Chip (ESP32-C3), Revision, MAC address, Flash (4MB), PSRAM, OS version
  - Two partition cards side by side: "OS Firmware" and "App Firmware" — active one highlighted in blue with "RUNNING" badge
  - "Scan for Boards" button when disconnected — opens a bottom sheet BLE scanner
  - Connected state: show board name + disconnect button in header
  - Show uptime counter (ticking up every second) when connected

  3. GPIO SCREEN (Tab 2 — "GPIO")
  This is the hero screen. Make it look like a professional embedded dev tool.
  - Pin grid: 2 columns, each pin is a card
  - Each card shows: GPIO number (large, colored), pin label, current state (HIGH=green dot, LOW=gray), mode badge (INPUT/OUTPUT/ADC)
  - OUTPUT pins: tap card to toggle HIGH/LOW — card fills green when HIGH
  - ADC pins: show a small horizontal bar showing value (0-4095) + numeric value
  - System pins (USB D+/D-, BOOT): shown with orange warning tint, cannot be toggled
  - Top filter bar: ALL | OUTPUT | INPUT | ADC — filter which pins show
  - Long-press any pin → bottom sheet to change mode (INPUT / OUTPUT / INPUT_PULLUP)
  - Animate state changes (quick flash on the card when value changes)
  - Show total pin count in header: "15 pins · ESP32-C3"

  4. TERMINAL SCREEN (Tab 3 — "Terminal")

    - Big "Flash →" button (red, destructive — requires confirm dialog)
    - Progress: animated gradient progress bar, percentage, estimated time remaining, "Do not disconnect" warning
    - After flash: success state with "Boot to App" CTA button
  - "Switch Partition" section: two buttons "Boot OS" / "Boot App" — tapping reboots to selected partition
  - History: last 3 flashed firmwares with name, size, date

  - Shows scanning animation (pulsing rings)

  - Only shows devices with name starting "ESP32-OS" (but in simulation shows a fake board)

  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Service UUID: e32f0001-b5a3-f393-e0a9-e50e24dcca9e

  CHAR_BOARD_INFO   e32f0002  READ       JSON: {chip, revision, mac, flash_mb, psram_mb, fw_version, app_partition}
  CHAR_GPIO_CONFIG  e32f0003  WRITE      JSON: {pin, mode}  — "INPUT"|"OUTPUT"|"INPUT_PULLUP"|"INPUT_PULLDOWN"
  CHAR_GPIO_WRITE   e32f0004  WRITE      JSON: {pin, value}  — 0 or 1
  CHAR_GPIO_STATE   e32f0005  READ+NOTIFY JSON: {pins: {"4":1,"5":0,...}, ts}
  CHAR_ADC_READ     e32f0006  WRITE+READ  Write: uint8 pin number. Read: uint16 ADC value
  CHAR_SERIAL_TX    e32f0008  WRITE       Raw bytes to ESP32 UART0
  CHAR_SERIAL_RX    e32f0009  NOTIFY      Raw bytes from ESP32 UART0
  CHAR_OTA_CTRL     e32f000a  WRITE+NOTIFY JSON ctrl: {cmd:"START",size,sha256} → notify {status:"READY"|"PROGRESS"|"DONE"|"ERROR",pct}
  CHAR_OTA_DATA     e32f000b  WRITE       Raw binary chunks (512B)
  CHAR_PARTITION    e32f0011  WRITE+READ  {cmd:"BOOT_OS"|"BOOT_APP"|"STATUS"} → {active:"os"|"app"}

  Wire these up in DeviceContext.tsx. When a real device is connected (not simulation), all calls go to actual BLE. When in simulation mode,
  calls update local state.
